<id>04-04-2</id>
<title>Implement Excel Generation Server Action</title>
<read_first>
- src/app/admin/attendance/actions.ts
- supabase_schema.sql
</read_first>
<action>
Add a new server action `exportAttendanceExcel(year: number)` in `src/app/admin/attendance/actions.ts`.
1. Verify admin session using `getSession()`.
2. Fetch all active employees from `employees` table: `select('id, name')`.
3. Fetch all attendance records for the given `year` from `attendance` table.
4. Import `exceljs` (`const ExcelJS = require('exceljs')` or `import ExcelJS from 'exceljs'`).
5. Construct a new Workbook: `const wb = new ExcelJS.Workbook()`.
6. Replicate the "Guide" sheet logic:
   - Name: "Guide"
   - Merged "Attendance System User Guide" (size 18, bold, color 1B4D4F).
   - Legend table with codes (P, A, L, HD, WO).
7. Loop from month 1 to 12 to create month sheets (e.g., "Jan", "Feb").
   - Header merged cells with "PRIMETEK GLOBAL SOLUTIONS" and "Attendance Register".
   - Setup columns for S.No, Employee Name, days 1-N, and summary columns.
   - For 50 rows, loop through available employees.
   - Populate weekends with "WO" and background FFEBEE.
   - Match attendance records from DB and fill the corresponding day cells with the status code ('P', 'A', 'L', 'HD').
   - Inject Excel formulas for the summary columns (`COUNTIF`, etc.).
8. Write workbook to buffer: `const buffer = await wb.xlsx.writeBuffer()`.
9. Return the buffer as a base64 string: `buffer.toString('base64')`.
</action>
<acceptance_criteria>
- `src/app/admin/attendance/actions.ts` exports `exportAttendanceExcel` function.
- The function generates a workbook and returns a base64 string.
</acceptance_criteria>
</task>
